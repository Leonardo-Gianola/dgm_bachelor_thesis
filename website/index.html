The Problem

The Darwin Gödel Machine (DGM) is an AI agent that rewrites its own source code to get better at solving real software engineering tasks. Each generation, it mutates its code, tests the mutation on actual GitHub issues, and keeps the change only if it helps.

It works. But it's expensive. Every mutation needs to be tested on real tasks inside Docker containers, costing time and money. The question this thesis asks:

Can we find equally good agents while spending significantly less?

* * *

The Approach

We compare four evaluation schedulers — different strategies for deciding how thoroughly to test each mutated agent. The core idea: don't waste expensive full evaluations on bad agents. Kill them early, spend the budget on promising ones.

Efficiency = resolved% / cost($)

The core metric. A scheduler that finds a 50%-accurate agent for $1 is five times more efficient than one that needs $5. This thesis argues efficiency — not raw accuracy — is what matters for practical self-improvement research.

More detail on the DGM loop

Step 1 — Seed. Start with a hand-crafted coding agent (20% accuracy on SWE-bench Verified Mini).

Step 2 — Select. Pick a parent agent from the archive of validated agents.

Step 3 — Mutate. A meta-LLM analyzes the parent's failures and proposes code changes — targeting prompts, tools, or workflow logic. Invalid Python is discarded immediately.

Step 4 — Evaluate. Test the mutant on real GitHub issues. The scheduler decides how many tasks to run and when to give up.

Step 5 — Archive. If the mutant improves on its parent, keep it. Otherwise discard. Repeat.

Results

The baseline agent started at 20% accuracy. After 5 generations and ~$5 in compute, its best child reached 50% on a 14-task medium-stage evaluation — a +30 pp improvement from a single targeted mutation to patch application logic.

Most generations produced weak or broken children. Gen 4 was the breakthrough — the only generation where a child survived past the small-stage filter and scored well on medium.

What We Found (So Far)

A single targeted mutation — fixing patch application logic — took the agent from 20% to 50% accuracy for ~$5 total. Most mutations broke things. The ones that worked were disproportionately impactful.

The comparative scheduler analysis — whether ASHA and Hyperband can find the same or better improvements more cheaply — is the primary remaining deliverable.

Full achievements & future work

Achievements:

  • 50% accuracy on medium-stage subset — up from 20% — in a single generation.
  • Four scheduling strategies with unified interface, compatible with original DGM.
  • Targeted mutations (patch logic) have disproportionate downstream effects.
  • Reproducible, budget-constrained framework with shared archive management.
  • Empirical validation as practical proxy for proof-of-improvement — works within $10.

Future Work:

  • Complete Hyperband, GA, and ASHA runs for comparative efficiency analysis.
  • Additional generations — does improvement compound, plateau, or oscillate?
  • Population-based training (PBT) with concurrent evolution branches.
  • Cross-benchmark transfer: SWE-bench → Polyglot without fine-tuning.
  • Safety constraints for self-modified agents.
